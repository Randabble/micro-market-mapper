### Model 1: The Spatio-Temporal "Event Pulse" Model
**Goal:** Specifically for high-volume event venues (Stadiums, Theaters).
**Logic:** Demand isn't a flat number; it’s a "Pulse" that decays as you move away from the venue.

**The Formula:**
$$AlphaScore = \left( \sum_{i=1}^{n} \frac{Capacity_i}{Dist_i^2} \right) \times \sqrt{S_{res}} \times (1 + \text{Event\_Status})$$

* **Variables:**
    * **$\sum \frac{Cap}{Dist^2}$:** Gravity model of demand. $Capacity$ of the venue divided by the square of the distance. If you are 2 blocks away, demand is 4x higher than if you are 4 blocks away.
    * **$S_{res}$:** Total residential driveway count in a 500m radius.
    * **Event_Status:** A binary multiplier (1 if an event is happening today, 0 if not).
* **Data Source:** Ticketmaster API for event schedules; Google Places for venue capacity.

### Model 2: The Economic Arbitrage & Friction Model
**Goal:** To identify areas where ParkPal has the highest "Price Advantage" over garages.
**Logic:** Drivers only switch to ParkPal if the **Savings > Friction** (walking).

**The Formula:**
$$AlphaScore = \left( \frac{Price_{Garage} - Price_{ParkPal}}{WalkTime_{min}} \right) \times \sqrt{D \times S_{res}}$$

* **Variables:**
    * **$Price_{Garage}$:** Current average hourly rate of the nearest 3 garages.
    * **$Price_{ParkPal}$:** Your target price (usually 50-60% of garage price).
    * **$WalkTime_{min}$:** Minutes to walk from the driveway to the destination.
* **Why it’s Robust:** This models **human behavior.** If a garage is $40 and your driveway is $20, but it’s a 15-minute walk, the "Arbitrage Score" is lower than a 5-minute walk.
* **Data Source:** SpotHero/API for garage prices; Google Matrix API for walking times.

### Model 3: The "Alpha V4" Machine Learning Feature Matrix
**Goal:** The ultimate, most robust model for a technical pitch.
**Logic:** Combine the Geometric Mean (Synergy) with **Zoning Risk** and **Safety Metrics**.

**The Formula:**
$$AlphaScore = \sqrt{D \times S_{res}} \times \left( \frac{Z + W + P_{pub}}{3} \right)$$

* **Variables:**
    * **$Z$ (Zoning Score):** 0 to 1 score based on whether the area allows Home Occupation Permits (from your Bellevue email research).
    * **$W$ (Walkability/Safety):** Use "Walk Score" or "Crime Map" data. High supply in a "scary" alley is worth zero.
    * **$P_{pub}$ (Public Scarcity):** The inverse of public parking availability.
* **The "Weighted Matrix" (How to code it):**
    Assign weights to these sub-metrics based on your user interviews (e.g., if drivers care more about safety than price, $W$ gets a higher weight).

```python
import math

class ParkPalEngine:
    def __init__(self, demand, supply, garage_price, walk_min, safety_score):
        self.D = demand          # 0-100 (from Ticket density)
        self.S = supply          # 0-100 (from Residential Parcel area)
        self.P_gar = garage_price # In USD
        self.W = walk_min        # Walking time to destination
        self.Safety = safety_score # 0-1 (1 is safest)

    def calculate_alpha_v4(self):
        # 1. Synergy Base (Geometric Mean)
        # Prevents "False Positives" where one side is zero
        synergy = math.sqrt(self.D * self.S)
        
        # 2. Economic Arbitrage Factor
        # People pay for convenience and savings
        target_price = self.P_gar * 0.6
        savings_per_min = (self.P_gar - target_price) / (self.W + 1) # +1 to avoid div by zero
        
        # 3. Final Multiplier (Safety & Scarcity)
        # We cap this at 1.2 to allow for a 'Surge' in perfect conditions
        multiplier = min(1.2, (self.Safety * 0.7) + (savings_per_min * 0.3))
        
        final_score = synergy * multiplier
        return round(final_score, 2)

# DEMO: Bellevue Square Area
# High Demand (90), Decent Supply (60), Garage is $30, 5 min walk, High Safety (0.9)
bellevue_market = ParkPalEngine(90, 60, 30, 5, 0.9)
print(f"Alpha V4 Score: {bellevue_market.calculate_alpha_v4()}")
```
